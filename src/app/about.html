This app was built using the following concepts from Angular 2:

  • Routing (provided by the Activated Route, Router, and Params components which is used in the Master/Detail view)
  • Modules (containing the needed components for http requests, among other features)
  • Services (custom service written to retrieve data from a local .json file)
  • Dependency Injection (to allow our custom data service to be used by modules/components throughout the app)
  • Components and Templates (four main components that encapsulate the logic, view (templates) and styles for each section of the app and keep them isolated)
  • Observables (used in the data retrieval service so that our views will update whenever new data is received from the external source)
  • Decorators: Injectable, Component, NgModule functions are used to add important meta-data to our classes, turning them into components.
  • Class syntax to create custom object types (FoodItem type is made available by exporting the custom FoodItem class)
  • Views (child views to display the food list (master and detail) and this about page)
  • Data-binding: One way binding of click events (HTML to JS app).

Additionally, the styles use:

  • Skeleton CSS framework (a light-weight CSS framwork containing a grid and various button and form styles)
  • Normalize CSS (included with Skeleton)
  • Imported Raleway font from Google.
  • Custom styles for colors (using Paletton tool), navigation, and hover effects
